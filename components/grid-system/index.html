The Lightning Design System grid, based on Flexbox, provides a flexible, mobile-first, device-agnostic scaffolding system. It includes helper classes that you can use to alter the look and behavior of your grid, such as alignment, order, flow, and padding helpers.

The grid system consists of two building blocks: the grid wrapper (.slds-grid) and the columns (.slds-col) that fill up the wrapper.

Grid Wrapper

To use the grid system, add the class .slds-grid to an element, component, or page layout. Grids can be nested, so you might have to override an inherited property. Some helper classes are available for this situation.

Adding the class .slds-wrap causes the flow of your .slds-col elements to wrap when they exceed 100% of their parent’s width. You can use .slds-nowrap to revert this behavior and allow your element to stretch and not wrap.

To stack your columns vertically instead of the default row behavior, use .slds-grid--vertical.

If you want your application to take 100% of the width and height of the viewport and nest other grids within, use the top-level app helper class .slds-grid--frame.

Grid Columns

By default, the grid columns use an equal amount of space in the parent container. This automatic sizing allows you to achieve most desired outcomes. However, you can add manual sizing classes to the columns if you need specific column widths.

Columns have no gutters. They are simply equal divisions of their parent. If you want gutters, remove the class .slds-col with the helper class .slds-col--padded, which adds a 0.75rem (12px) gutter to the left and right side of your column.

Using the manual sizing class helpers, you can specify a column span across the following grids –2, 3, 4, 5, 6, and 12. The grid supports up to 12 columns.

See sizing utilities for more information and examples.

Vertical Alignmentdev ready

The grids have some handy helpers that allow you to specify the vertical placement of content on the main axis. You can apply .slds-align-top, slds-align-middle, and .slds-align-bottom to a column within a grid wrapper to change the alignment of the content within.

Orderdev ready

If you need to keep your DOM in an order different from the application’s visual display, you can add helper classes to adjust the source order within your grid wrapper. A basic usage is to apply the class .slds-order--1 to an element that’s lower in the DOM tree and then add .slds-order--2 to a top-level DOM element.
